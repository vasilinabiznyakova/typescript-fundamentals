Спец типы - 4 

- any
 для начинающих друг для проектов враг, отключает полностю любую проверку типов, не рекомендуется использовать в реальных проектак тк убивает всю эффективность ts. noImplicit any есть опция в ts config если тип не установлен то ts присвоит any если отключить опции явной ошщибки не будет но лучше не отключать compilerOptions>strict>false
В продакшн проектах не стоит его использовать в 99%

Any одновременно является надтипом и подтипом для всего

Подсказка методов также работать не будет

- unknown

вынуждает нас делать проверки на типы например через typeof используя их при этом безопасно,то есть это анвлог any но безопасный.
Если мы не знаем какой тип ожидается на вход мы можем обьявить его unknown + использовать проверки на типы и безопасно проверять

unknown - является супертипом для всего но не может быть подтипом ни одного другого типа то есть присвоить строке  unknown нельзя КРОМЕ самого себя и any

- never
является подтипом всех других типов, это пустое множество пародокс всех подтипов, то что не может существовать

Если функция завершается с ошибкой она всегда вернет never или если она содержит бесконечный цикл и никогда не завершиться

- void
тип который означает что функция ничего не возвращает
